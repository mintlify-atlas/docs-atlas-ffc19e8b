# Form

Form is a wrapper component that provides a semantic `<form>` element with automatic spacing between form fields. It ensures consistent layout and accessibility for form structures.

## Installation

```bash
yarn add @twilio-paste/core
```

Or if you need just this component:

```bash
yarn add @twilio-paste/form
```

## Usage

```jsx
import { Form } from '@twilio-paste/core/form';
import { Label } from '@twilio-paste/core/label';
import { Input } from '@twilio-paste/core/input';
import { Button } from '@twilio-paste/core/button';

const MyComponent = () => {
  const handleSubmit = (e) => {
    e.preventDefault();
    // Handle form submission
  };

  return (
    <Form onSubmit={handleSubmit}>
      <Label htmlFor="email">Email</Label>
      <Input id="email" name="email" type="email" required />
      
      <Label htmlFor="password">Password</Label>
      <Input id="password" name="password" type="password" required />
      
      <Button type="submit" variant="primary">Sign In</Button>
    </Form>
  );
};
```

### Form with Validation

```jsx
import { Form } from '@twilio-paste/core/form';
import { FormControl } from '@twilio-paste/core/form-control';
import { Label } from '@twilio-paste/core/label';
import { Input } from '@twilio-paste/core/input';
import { HelpText } from '@twilio-paste/core/help-text';
import { Button } from '@twilio-paste/core/button';

const [errors, setErrors] = React.useState({});

<Form onSubmit={handleSubmit}>
  <FormControl>
    <Label htmlFor="phone" required>
      Phone Number
    </Label>
    <Input
      id="phone"
      name="phone"
      type="tel"
      required
      hasError={!!errors.phone}
    />
    {errors.phone ? (
      <HelpText variant="error">{errors.phone}</HelpText>
    ) : (
      <HelpText>Include country code</HelpText>
    )}
  </FormControl>

  <Button type="submit" variant="primary">
    Submit
  </Button>
</Form>
```

## Props

<ParamField path="children" type="ReactNode" required>
  Form controls and content.
</ParamField>

<ParamField path="maxWidth" type="BoxStyleProps['maxWidth']">
  Maximum width of the form container. Use Paste size tokens (e.g., "size60", "size70").
</ParamField>

<ParamField path="element" type="string" default="FORM">
  Overrides the default element name to apply unique styles with the Customization Provider.
</ParamField>

<ParamField path="onSubmit" type="FormEventHandler">
  Form submission handler.
</ParamField>

Form also accepts all standard HTML form attributes (action, method, encType, etc.).

## Spacing

Form automatically applies vertical spacing between child elements:

- Default gap: `space80` (1.25rem / 20px)
- Creates consistent spacing without manual margin management
- Uses CSS flexbox with column direction

## Best Practices

### Do

- Use Form as the container for all form-related inputs
- Include proper labels for all form fields
- Use FormControl to group labels with their inputs
- Handle form submission with onSubmit
- Provide validation feedback with error messages
- Use appropriate input types for better mobile UX
- Set maxWidth for better readability (typically size60-size70)

### Don't

- Don't use Form for non-form content
- Don't forget to handle form submission (preventDefault)
- Don't skip labels or rely on placeholder text alone
- Don't create overly long forms - break into steps if needed
- Don't use only client-side validation - always validate server-side
- Don't forget to indicate required fields

## Accessibility

- Uses semantic `<form>` HTML element
- Properly associates labels with inputs
- Supports keyboard navigation (Tab, Enter to submit)
- Error messages are associated with their fields
- Required fields are properly marked
- Focus management on validation errors
- Ensure submit buttons are clearly identified

### Form Validation

- Use HelpText with variant="error" for validation messages
- Set hasError prop on invalid inputs
- Move focus to first error on submission
- Announce errors to screen readers
- Show inline validation when appropriate
